# 如何访问 Zeabur 上的服务

## 方法 1：查找自动分配的域名

### 步骤 1：在 Zeabur 控制台中找到域名

根据你的截图，你现在在"网络"标签页。请按照以下步骤操作：

1. **在左侧项目列表中**，点击 `pbl-learning` 服务（如果还没选中）

2. **查看顶部标签栏**，你应该能看到：
   - 概览
   - 环境变量
   - 变量组
   - 日志
   - **网络** ← 你现在在这里
   - 构建
   - 设置

3. **在"网络"标签页中**：
   - 查看是否有"域名"或"Domain"部分
   - 如果有显示域名（如 `xxx.zeabur.app`），直接复制使用
   - 如果没有域名，需要添加一个

### 步骤 2：添加域名（如果还没有）

如果在"网络"标签页中没有看到域名，你需要添加一个：

1. **点击"生成域名"或"添加域名"按钮**
   - Zeabur 会自动为你生成一个免费域名
   - 格式类似：`pbl-learning-xxx.zeabur.app`

2. **等待域名生成**（通常几秒钟）

3. **复制生成的域名**

### 步骤 3：在浏览器中访问

复制域名后，在浏览器中访问：

```
https://你的域名.zeabur.app
```

例如：
```
https://pbl-learning-abc123.zeabur.app
```

## 方法 2：从服务概览页面查看

1. **点击顶部的"概览"标签**
2. 在概览页面中，通常会显示：
   - 服务状态（Running/Stopped）
   - 访问地址/域名
   - 资源使用情况

3. **找到"访问地址"或"URL"部分**，复制域名

## 方法 3：检查服务是否正在运行

在访问之前，确保服务正在运行：

### 检查服务状态

1. **查看服务状态指示器**：
   - ✅ 绿色/Running = 服务正常运行
   - 🔄 黄色/Building = 正在构建部署
   - ❌ 红色/Failed = 部署失败

2. **如果服务未运行**：
   - 点击"日志"标签查看错误信息
   - 检查构建日志是否有错误

## 常见问题

### Q1: 找不到域名选项

**可能原因**：
- 服务还在构建中
- 服务部署失败
- 需要手动添加域名

**解决方法**：
1. 检查服务状态是否为 "Running"
2. 查看"日志"标签，确认部署成功
3. 在"网络"标签中点击"添加域名"或"生成域名"

### Q2: 访问域名显示 404 或无法访问

**可能原因**：
- 服务刚启动，DNS 还在传播（等待 1-2 分钟）
- 服务端口配置错误
- 服务启动失败

**解决方法**：
1. 等待 1-2 分钟后重试
2. 检查"日志"标签，查看服务是否成功启动
3. 确认看到类似这样的日志：
   ```
   ✓ Ready in XXXms
   ✓ Compiled successfully
   - Local: http://localhost:3000
   ```

### Q3: 域名可以访问，但页面显示错误

**可能原因**：
- 环境变量未配置
- Supabase 连接失败（本地地址无法访问）

**解决方法**：
1. 点击"环境变量"标签
2. 确认已配置：
   - `NEXT_PUBLIC_SUPABASE_URL`
   - `NEXT_PUBLIC_SUPABASE_ANON_KEY`
3. 如果使用本地 Supabase（`http://10.1.20.75:8000`），需要改用云服务

## 快速测试步骤

一旦获得域名，按以下顺序测试：

### 1. 测试首页
```
https://你的域名.zeabur.app
```
应该看到 "PBL Learning Platform" 标题

### 2. 测试登录页
```
https://你的域名.zeabur.app/login
```
应该看到登录表单

### 3. 测试 Dashboard
```
https://你的域名.zeabur.app/dashboard
```
未登录会重定向到登录页

### 4. 测试 AI 聊天
```
https://你的域名.zeabur.app/chat
```
应该看到聊天界面
